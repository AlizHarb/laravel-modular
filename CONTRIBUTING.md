# Contributing

Thank you for considering contributing to Laravel Modular!

## Security Vulnerabilities

If you discover a security vulnerability within Laravel Modular, please send an e-mail to Aliz Harb via harbzali@gmail.com. All security vulnerabilities will be promptly addressed.

## Pull Requests

- **Branching**: Create a feature branch for your changes (`git checkout -b feature/amazing-feature`).
- **Tests**: Ensure all tests pass (`vendor/bin/pest`).
- **Pint**: Format your code using Laravel Pint (`vendor/bin/pint`).
- **PHPStan**: Ensure no new static analysis errors are introduced (`vendor/bin/phpstan`).
- **Documentation**: Update the README or create a PR with documentation changes if applicable.

## Coding Standards

Laravel Modular follows PSR-12 and standard Laravel coding conventions. Please ensure your code matches the existing style.
